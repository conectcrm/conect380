# GDN-501 - Go live checklist Guardian

## 1. Preparacao pre-janela
- [ ] Sprint 4 gate aprovado (`GDN-410`).
- [ ] Runbook de incidente e matriz N1/N2/N3 publicados.
- [ ] Monitoramento de billing/plataforma validado.
- [ ] Drill de rollback validado.
- [ ] Treinamento N1/N2/N3 concluido.

## 2. Preflight tecnico
- [ ] Flags de transicao legado validadas.
- [ ] Smoke de operacao Guardian validado.
- [ ] Replay/recovery de webhook validado.
- [ ] Observabilidade de alertas validada.
- [ ] Carga leve validada.

## 3. Janela de go-live
- [ ] Inicio de janela registrado.
- [ ] Responsaveis on-call confirmados.
- [ ] Comunicacao de inicio enviada.
- [ ] Monitoramento continuo ativo.
- [ ] Incidentes registrados e tratados pelo runbook.

## 4. Criterio de GO
- [ ] Nenhum incidente P0/P1 aberto.
- [ ] Fluxos criticos operacionais sem regressao.
- [ ] Indicadores de billing sob controle.
- [ ] Auditoria critica registrando eventos.

## 5. Criterio de NO-GO
- [ ] Falha critica sem mitigacao no tempo alvo.
- [ ] Erro sistemico em billing com risco financeiro.
- [ ] Ruptura de seguranca/governanca administrativa.

## 6. Encerramento da janela
- [ ] Decisao final registrada (GO/NO-GO).
- [ ] Evidencias anexadas.
- [ ] Acoes pos-go-live publicadas.
